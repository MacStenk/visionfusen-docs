# Willkommen bei VisionFusen

**Medien-Pipeline für das neue Internet.**

Normaler Web-Content rein. Signierter, indexer-ready Output raus. Die Brücke zwischen Web2-Publishing und Web3-Ownership.

## Das Problem

Content Discovery ist der größte Pain Point im dezentralen Web. Wer hat was wann erstellt? Wie wird Content gefunden? Wie monetarisiert man ohne Plattform-Lock-in?

- **Ownership** — Wer hat das Original erstellt?
- **Discovery** — Wie finden Menschen (und Maschinen) deinen Content?
- **Monetarisierung** — Wie verdienst du ohne Mittelsmänner?

## Die Lösung: VisionFusen Pipeline

```
Content → Optimierung → Metadaten → Signatur → Distribution
```

### 1. Optimierung
WebP/AVIF, saubere Struktur, komprimiert für Edge-Delivery.

### 2. Metadaten
Schema.org, JSON-LD — maschinenlesbar für LLMs und Indexer.

### 3. Signatur
NIP-94 (Bilder), NIP-23 (Texte) — kryptografischer Beweis auf Nostr.

### 4. Distribution
Gleichzeitig auf Nostr Relays und klassischem Web.

## LLMO — LLM Optimization

Nicht nur für Menschen lesbar, sondern für Maschinen verstehbar.

- **Schema.org** — Strukturierte Daten
- **JSON-LD** — Maschinenlesbare Metadaten
- **llms.txt** — Direkter Zugang für AI-Crawlers
- **Citation-Optimized** — Deine Seiten tauchen als Quelle in KI-Antworten auf

## Drei Monetarisierungs-Layer

| Layer | Zielgruppe | Mechanismus |
|-------|------------|-------------|
| **Human** | Web-Besucher | Paywall via LNbits, Pay-per-Post |
| **Nostr** | Community | Value-for-Value, Zaps |
| **Agent** | AI-Agenten | L402 API, Sats per Call |

## Für wen?

- **Content Creator** — Sichere Ownership, mehrere Monetarisierungswege
- **Coaches & Consultants** — Signierte Materialien, eigene Infrastruktur
- **Entwickler** — API-First, selbst hostbar, erweiterbar
- **Early Adopters** — Bau mit am dezentralen Content-Stack
